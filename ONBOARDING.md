# Как подключить новую клинику

Пошаговый гайд. Занимает ~15 минут, если Evolution API уже поднят.

---

## Шаг 1. Поднять инстанс Evolution API

> Пропусти, если сервер Evolution уже запущен.

```bash
# Запустить Evolution API (один раз на сервер)
docker run -d --name evolution -p 8080:8080 \
  -e AUTHENTICATION_API_KEY=<твой-глобальный-ключ> \
  atendai/evolution-api:latest
```

Создать инстанс для клиники (имя должно быть уникальным на сервере):

```bash
curl -X POST http://localhost:8080/instance/create \
  -H "apikey: <твой-глобальный-ключ>" \
  -H "Content-Type: application/json" \
  -d '{"instanceName":"clinic_имя","integration":"WHATSAPP-BAILEYS"}'
```

Привязать номер WhatsApp по QR-коду:

```bash
# Открыть в браузере — покажет QR для сканирования
http://localhost:8080/instance/connect/clinic_имя
```

На телефоне клиники: WhatsApp → Связанные устройства → Привязать устройство → сканировать QR.

Прописать webhook на наш приёмник:

```bash
curl -X POST http://localhost:8080/webhook/set/clinic_имя \
  -H "apikey: <твой-глобальный-ключ>" \
  -H "Content-Type: application/json" \
  -d '{
    "webhook": {
      "enabled": true,
      "url": "https://<наш-хост>/webhook/whatsapp/?token=<WHATSAPP_WEBHOOK_TOKEN>",
      "events": ["MESSAGES_UPSERT"]
    }
  }'
```

Значение `WHATSAPP_WEBHOOK_TOKEN` берётся из `.env` нашего сервера.

---

## Шаг 2. Создать Clinic в Django Admin

Открыть: `https://<наш-хост>/admin/clinics/clinic/add/`

Заполнить обязательные поля:

| Поле | Описание | Пример |
|------|----------|--------|
| **Название** | Имя клиники (попадёт в системный промпт) | `Жемчуг Дент` |
| **Номер WhatsApp** | Номер-получатель в формате E.164 (без `+`) | `77001112233` |
| **Инстанс Evolution** | Имя инстанса из Шага 1 — точное совпадение | `clinic_имя` |
| **Часовой пояс** | IANA-таймзона для корректного разбора дат | `Asia/Almaty` |
| **Активна** | Галочка: только активные клиники обрабатываются | ✓ |

## Шаг 3. Заполнить прайс, часы и менеджера

**Услуги и цены** (`services_json`) — JSON-массив:

```json
[
  {"name": "Осмотр и консультация", "price": "3 000 ₸"},
  {"name": "Профессиональная чистка", "price": "14 000 ₸"},
  {"name": "Лечение кариеса", "price": "от 18 000 ₸"}
]
```

**Часы работы** (`working_hours`) — JSON-объект:

```json
{
  "Пн–Пт": "09:00–20:00",
  "Сб": "10:00–18:00",
  "Вс": "выходной"
}
```

**Уведомления о заявках** — в разделе «Уведомления о заявках»:

| Поле | Описание |
|------|----------|
| **WhatsApp менеджера** | Номер (E.164) для уведомлений о новых заявках на запись |
| **Уведомления включены** | Если выключено — заявки создаются, но менеджер не получает сообщений |

> Бот **никогда не подтверждает запись сам** — он только собирает заявку и передаёт
> менеджеру. Подтверждение — всегда решение человека.

## Шаг 4. Проверить готовность

```bash
docker compose exec web python manage.py check_clinic <instance_name>
```

Пример вывода для готовой клиники:

```
──────────────────────────────────────────────────────
  Клиника: Жемчуг Дент  (id=18)
──────────────────────────────────────────────────────

  ✓  Активна (is_active=True)
  ✓  Инстанс задан: demo_clinic_a
  ✓  Номер WhatsApp: +77010000001
  ✓  Услуги заполнены: 5 позиций
  ✓  Часы работы: Пн–Пт, Сб, Вс
  ✓  Адрес: г. Алматы, ул. Назарбаева, 45, офис 102
```

не найденных по `instance_name`.

## Шаг 5. Отправить тестовое сообщение

С любого номера отправить в WhatsApp клиники вопрос, например:
- «Сколько стоит чистка зубов?»
- «Во сколько вы работаете в субботу?»
- «Хочу записаться на приём»

Убедиться, что:
1. Бот ответил (не молчит).
2. Цена/часы из его ответа совпадают с тем, что заполнено в админке.
3. При запросе на запись — бот задаёт уточняющие вопросы, а не подтверждает приём сразу.
4. Менеджер получил уведомление в WhatsApp (если `manager_whatsapp` задан).

## Частые проблемы

| Симптом | Вероятная причина |
|---------|-------------------|
| Бот молчит на входящее | `is_active=False`, неверный `instance_name`, webhook не настроен |
| Бот отвечает данными другой клиники | `instance_name` в БД не совпадает с именем инстанса Evolution |
| Уведомление менеджеру не приходит | `manager_whatsapp` пуст или `notifications_enabled=False` |
| `check_clinic` выдаёт «не найдена» | `instance_name` в БД пуст или написан с опечаткой |
| Evolution не отправляет сообщения | Проверить `EVOLUTION_API_URL`, `EVOLUTION_API_KEY` в `.env` |

## Переменные окружения (`.env`)

Для Evolution необходимы три переменные:

```env
WHATSAPP_PROVIDER=evolution
EVOLUTION_API_URL=http://evolution:8080   # URL Evolution API
EVOLUTION_API_KEY=<глобальный-ключ>       # Заголовок apikey
EVOLUTION_INSTANCE=                        # Оставить пустым при мультитенанте —
                                           # каждая клиника использует свой instance_name из БД
```

> `EVOLUTION_INSTANCE` нужен только как запасной вариант для старых клиник без
> `instance_name` в БД. При мультитенанте лучше задать `instance_name` каждой клинике
> и оставить эту переменную пустой.
